# 双人播客脚本评分 Rubric（"市场可用"的定义）

> `duo-script-judge` 的评分真源。10 维，每维 1–5 分。打分要严格、要给证据（引用脚本原文 + 源文对照）。
> 目的：把"市场可用"从主观感觉变成可量化、可迭代的门槛。
> **双人专属**：新增 Dialogue Chemistry（对话化学反应）+ Role Balance（角色平衡）两个维度，取代单人重复项；其它维度评分标准适配双人对话。

## 评分维度

### 1. 钩子 Hook / Cold Open（前 ~20 秒，双人建立互动）
- **5**：cold open 双人快速入题（A 抛反问/痛点/悬念/场景 → B 立刻接话追问/补充/吐槽），15 秒内落钩并建立"两人在聊"的感觉，零问候套话；落钩后自然报家门（品牌+主题，不报集数），谁说不生硬。
- **4**：有效双人 cold open，吸引人，仅微瑕（如 B 接话略慢）。
- **3**：有钩子意图但平淡，或 A 一个人说太久才轮到 B，或报家门位置生硬。
- **2**：开场啰嗦/慢热，或像采访式"A 问 B 答"，前 15 秒抓不住人。
- **1**：问候式/公式化开场（"大家好/欢迎/今天聊"先于内容），或 A 独白开场。

### 2. 结构 Arc
- **5**：hook→promise→body→climax→CTA 完整且自然，要点全部对话推进、结论先行。
- **4**：结构清晰，偶有一处过渡略硬。
- **3**：大体有结构，但缺承诺/高潮或要点不结论先行。
- **2**：流水账，无明显弧线。
- **1**：结构混乱/断裂。

### 3. 结尾 Outro
- **5**：清晰回顾要点 + 一个有力 takeaway + 一个明确 CTA（订阅/关注）+ 双人对句自然告别，与正文呼应，零套话。
- **4**：收尾有效，仅微瑕。
- **3**：有收尾但偏平淡/套路，或缺回顾、缺 takeaway/CTA。
- **2**：收尾仓促，或套话（"今天就聊到这里""感谢收听""我们下期再见"品牌定式）。
- **1**：无收尾/硬断。

### 4. 趣味性 Fun
- **5**：比喻/类比 ≥2、口语夸张/生动表达 ≥2、**双主持观点交锋/接梗/吐槽 ≥2**、有人设观点、用具体场景落地；听着不困且不油腻，趣味服务内容。
- **4**：有趣味元素，密度/自然度偶有不足。
- **3**：少量趣味，偏"讲知识"，枯燥段较多。
- **2**：几乎无比喻/生动表达，干瘪。
- **1**：满屏说教/念稿，无任何趣味。

### 5. 反套路 Anti-formulaic
- **5**：零 AI 套路——无三段及以上排比收尾（含"A/B 一唱一和的工整排比"）、无路标编号、无套式连接词反复、无"反认知/反直觉"等 AI 高频词；句式错落。
- **4**：基本无套路，仅 1 处 borderline。
- **3**：出现 1-2 处明显套路（如一处排比收尾、或一处路标编号、或两人对句工整对仗）。
- **2**：多处套路，AI 味明显。
- **1**：通篇模板化/工整排比/路标化，AI 味浓。

### 6. 口语化 Orality（双人口语一致）
- **5**：两主持全程口语化一致（同一个"朋友圈"的语感）、短句、口语过渡，零机器味词，无 Markdown，段落全 ≤80 字，角色标注格式正确。
- **4**：口语化好，仅 1 处机器味残留或个别长段。
- **3**：偏书面，有少量机器过渡词或长句。
- **2**：明显书面化/朗读感，多处机器词。
- **1**：基本是文章朗读，机器味浓。

### 7. 对话化学反应 Dialogue Chemistry（**双人专属**）
- **5**：有来有回自然真实——B 的回应**针对 A 刚说的内容**（引用/转述/反驳/补充），每 3-5 轮有观点小交锋，无各说各话、无机械一问一答、无一人独白一人捧哏。
- **4**：对话自然，偶有一处回应略泛（如"嗯对"式接场）。
- **3**：部分对话真实，但有明显的"各说各话"或"采访式一问一答"段。
- **2**：大量各说各话/机械问答，不像真人对聊。
- **1**：基本是两段独白拼在一起，无互动。

### 8. 角色平衡 Role Balance（**双人专属**）
- **5**：A/B 戏份/字数大致均衡（字数占比 ∈ [40%, 60%]），轮换频繁（总轮次 ≥18），无连续同角色 >2 段，两主持都既推进又补充（无机械分工）。
- **4**：基本均衡（占比 ∈ [35%, 65%]，轮次 ≥15），偶有一人连续 3 段。
- **3**：略显失衡（占比接近 35%/65% 边界，或轮次 12-14，或一人连续 >3 段）。
- **2**：明显失衡（一人占 >65%，或轮次 <12，或一人长独白）。
- **1**：严重失衡（基本是独白，另一人只是点缀）。

### 9. 信息保真 Fidelity（**硬门禁**）
- **5**：所有事实 100% 可回溯源文，核心观点全覆盖，零虚构（双人对白里的所有事实都核对）。
- **4**：可回溯，核心观点覆盖，仅措辞口语化无失真。
- **3**：有个别无法核实/轻微夸大的表述。
- **2**：有编造/源文外事实/数据失真（含 B 自己编数据"补充"）。
- **1**：大量虚构。
- **注**：Fidelity ≤3 直接判不合格，无论其它维度多高。

### 10. TTS 友好 + 元数据 TTS & Metadata
- **5**：短分句、断句自然、无密集数字/术语堆砌；**每个 turn 语义完整**（无半句话被拆到两个角色）；标题≤35字、**含数字钩子（优先源文事实数字）**、关键词前置、**前10字无英文堆砌**、不带集号前缀；简介200-300字（前100字含核心关键词+具体事实）；标签5-8。
- **4**：基本友好，仅个别密集串需拆或数字钩子偏弱。
- **3**：有几处 TTS 易吞字/读错的长串，或 turn 边界有半句话；或元数据某项不达标。
- **2**：多处数字/术语堆砌预期吞字严重，或元数据多项不达标。
- **1**：满屏密集信息串 TTS 必崩，或元数据缺失/严重不合格。

## 市场可用门槛（market-ready bar）

- 每个维度 ≥ **4**（10 维：hook / structure / outro / fun / anti_formulaic / orality / dialogue_chemistry / role_balance / fidelity / tts_metadata）
- 总分（10 维平均）≥ **4.0**
- 无任何维度 < 4（即不许有短板）
- **Fidelity 必须零虚构**（Fidelity=5，且 judge 的 `fabricated_claims` 为空）

未达标 → 进入 fix 循环，针对低分维度定向改写。

```json
{
  "round": 1,
  "market_ready": false,
  "overall": 3.9,
  "dimensions": {
    "hook": { "score": 5, "evidence": "引用脚本原文片段（含角色标注）", "gaps": [] },
    "structure": { "score": 4, "evidence": "...", "gaps": ["高潮段偏弱"] },
    "outro": { "score": 4, "evidence": "A:回顾要点→B:订阅引导对句收尾", "gaps": [] },
    "fun": { "score": 4, "evidence": "比喻'像把说明书读给朋友听'+交锋'A:快得离谱→B:真的假的'", "gaps": [] },
    "anti_formulaic": { "score": 5, "evidence": "无排比收尾/无路标编号/无A-B工整对仗", "gaps": [] },
    "orality": { "score": 3, "evidence": "B第3段出现'首先'", "gaps": ["去机器词：B第3段"] },
    "dialogue_chemistry": { "score": 4, "evidence": "B 多次转述 A 的话再回应，有2处观点交锋", "gaps": [] },
    "role_balance": { "score": 4, "evidence": "A 48% / B 52%，轮次17", "gaps": [] },
    "fidelity": { "score": 5, "evidence": "逐条对照源文（双人对白）", "fabricated_claims": [] },
    "tts_metadata": { "score": 3, "evidence": "第6段'B:28到99秒中位40到50秒'密集数字串", "gaps": ["拆第6段数字串"] }
```
