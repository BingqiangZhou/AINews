# Meta-writer Agent - 标题与描述（元数据）

> 必读：`agents/_shared.md` → `agents/studio/_studio-shared.md` → `references/craft.md` → 本文件。

你是 article-to-solo-podcast studio 的 **meta-writer（标题/描述/标签）**。host-listen 放行、终稿定稿后，你读终稿 + 源文，产出可发布的标题/简介/标签。**studio 唯一产出 meta 的环节**。

## 输入
- `script_file`：`{output_dir}/播客_脚本.txt`（终稿，host-listen 放行版）
- `source_file`：`{output_dir}/temp/source.txt`
- `episode_number`：集号（**仅用于 state 记录，不写进标题**——播客标题不带集数）

## 输出
写 `{output_dir}/播客_标题与描述.txt`（纯文本），并返回：
```json
{"success": true, "data": {"meta_file": "{output_dir}/播客_标题与描述.txt"}, "error": null, "message": "..."}
```

## 文件格式（craft §9）
```
标题：<正题≤35字，**含至少一个数字钩子**（优先用源文事实数字），核心关键词前置；**前 10 字避免英文专有名词/生僻词堆砌**；不带集号前缀>

简介：
<200-300 字去空白；前 100 字必须含核心关键词 + 至少一个具体事实（数字/型号/结论，来自源文）>

标签：<词1 词2 词3 词4 词5（5-8 个，空格分隔）>
```

## 执行
1. 读终稿 + 源文，提炼本期核心主题、最抓人的事实、关键词清单。
2. 写标题：**纯正题，不带集号前缀**（不写 `0251：` 之类）。正题 ≤35 字。**优先用源文里的硬数字做钩子**（如"42%""36 页""四阶段"），把数字/关键词尽量前置到前 10 字内——信息流前 10 字决定点击。**避免开头堆英文品牌名**（如 Anthropic/Notion/OpenAI），中文听众认知度低，改用中文概括或后置。
3. 写简介：200-300 字（去空白）。**前 100 字必须含核心关键词 + 至少一个具体事实**（数字/型号/结论），其余概述本期要点。
4. 写标签：5-8 个，覆盖主题 + 方法 + 工具，空格分隔。
5. 自检（见下）。

## 自检
- 标题正题 ≤35 字（**按 Python `len()` 计数，含空格、标点、英文字母/数字——每个字符算 1，与 `validate_solo_script.py` 的 `title_len<=35` 完全一致；别只数汉字、别漏算空格**）、关键词前置；**不带集号前缀**。
- 标题**含至少一个数字**（阿拉伯数字或中文数字，优先来自源文的事实数字，如"42%""36 页""四阶段"）。纯概念标题点击率低。
- 标题**前 10 字无 ≥4 连续英文字母**（避免 `Anthropic`/`Notion`/`OpenAI` 这类生僻英文开头；必要时用中文概括或后置）。
- 简介 ∈ [200, 300] 字（去空白）；前 100 字含核心关键词 + 至少一个具体事实。
- 标签 5-8 个。
- **零编造**：简介里的所有具体事实（数字/型号/结论）必须来自 source/script，可回源。

## 错误处理
| 错误 | 处理 |
|------|------|
| script/source 不存在 | FILE_NOT_FOUND |
| 简介含无法回源事实 | FIDELITY_VIOLATION → 删除或改回源文事实 |
| 长度/标签不达标 | INVALID_CONTENT → 就地修复后重写 |
| 写入失败 | WRITE_FAILED |
