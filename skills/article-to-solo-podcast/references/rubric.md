# 单人资讯播客脚本评分 Rubric（"市场可用"的定义）

> `solo-script-judge` 的评分真源。10 维，每维 1–5 分。打分要严格、要给证据（引用脚本原文 + 源文对照）。
> 目的：把"市场可用"从主观感觉变成可量化、可迭代的门槛。
> **专业资讯播报取向（快报 + 深读双层结构）**：评分锚定"准确、清晰、克制、信息密度高 + 双层节奏"，不奖励闲聊腔/夸张/人设。第三人称客观陈述是资讯播报的正常腔调，不扣分。脚本应包含**快报（多条简讯）+ 今日深读（1 条深讲）**双层结构，深读给听众一个记忆锚点。若当日素材不足以支撑深读（源文所有要闻都信息量单薄），允许全快报模式（不扣分，但需 state 标 degraded）。

## 评分维度

### 1. 钩子 Hook / 开场（前 ~30 秒）
- **5**：开场直接点出当日最重磅事实 + 一句重要性定调，15 秒内入题，零问候套话；定调后自然报家门（品牌+集号+主题），不生硬；**报家门后给路线图**——预告每个板块的**亮点条目**（不只是板块名）+ 点出今日深读主题，给听众"想听完"的理由。
- **4**：有效开场，吸引人，仅微瑕（如路线图只列板块名未给亮点条目，或未预告深读主题）。
- **3**：有入题意图但平淡，或报家门位置生硬（如开头立刻报家门未先给事实），或缺路线图。
- **2**：开场啰嗦/慢热，前 15 秒抓不住要点，无路线图。
- **1**：问候式/公式化开场（"大家好/欢迎/今天聊"先于内容）。

### 2. 结构 Arc（快报 + 深读 双层）
- **5**：开场→路线图→快报（多条简讯，对应文章章节，结论先行，节段切换有明确信号）→今日深读（1 条深讲，标 `[SECTION:DEEPDIVE]`，含背景/数据 + 三维分析 + 反向风险）→收束小结→CTA，完整且自然。双层节奏清晰，深读给听众记忆锚点。**注**：若当日素材不足（state 标 `no_deepdive_material`），全快报模式结构完整亦可 5 分。
- **4**：结构清晰，偶有一处切换信号略弱，或深读缺反向风险批判。
- **3**：大体有结构，但缺路线图/收束，或要点不结论先行，或深读沦为普通快报（无展开）。
- **2**：流水账念清单，无明显弧线，无深读环节（且素材本可支撑）。
- **1**：结构混乱/断裂。

### 3. 结尾 Outro
- **5**：清晰收束脉络（"今天 X 条要闻，核心是…"）+ （有合适话题时）一个轻互动钩子（如"那么聊到这儿，也想问问你……"）+ 一个明确 CTA（订阅/关注明日日报）+ 自然告别，与正文呼应，零套话。
- **4**：收尾有效，仅微瑕（如缺互动钩子，但脉络小结和 CTA 到位）。
- **3**：有收尾但偏平淡/套路，或缺脉络小结或缺 CTA。
- **2**：收尾仓促，或套话（"今天就聊到这里""感谢收听""我们下期再见"品牌定式）。
- **1**：无收尾/硬断。

### 4. 清晰度 Clarity（取代旧"趣味性 Fun"维度）
- **5**：资讯条目结论先行；专业术语（MoE/Scaling Law 等）首次出现有通俗解释或上下文；信息密度高但不堆砌；逻辑链清晰，非专业听众跟得上；**自然出现的单点类比/口语锚点**能帮抽象概念落地（加分项，不作强制）。
- **4**：基本清晰，仅个别术语未解释或个别条目结论不够前置。
- **3**：部分术语生硬未解释，或信息组织略乱，听众需回放。
- **2**：术语堆砌无解释、信息混乱、多数条目抓不住重点。
- **1**：满屏术语/数字堆砌，听众完全跟不上。
- **注**：本维度**不考核夸张/人设观点**，但**鼓励自然类比/口语锚点**（能讲清抽象概念的单点类比可加分）。禁满屏比喻——过度类比反扣分。

### 5. 反套路 Anti-formulaic
- **5**：零 AI 套路——无三段及以上排比收尾、无套式连接词反复、无"反认知/反直觉/值得注意"等 AI 高频词；句式错落。
- **4**：基本无套路，仅 1 处 borderline。
- **3**：出现 1-2 处明显套路（如一处排比收尾、或一处套词反复）。
- **2**：多处套路，AI 味明显。
- **1**：通篇模板化/工整排比，AI 味浓。
- **注**：节段切换词（"先说…/再看…/最后是…"）是资讯播报的结构信号，**不视为套路**，但不要连续多节用同一个。

### 6. 口语化 Orality（TTS 友好的播报腔）
- **5**：TTS 友好的短句、自然停顿、断句合理，零机器味词，无 Markdown，段落全 ≤80 字；播报腔自然流畅（第三人称客观陈述为正常态）；**信源自然嵌入**（"据 XX 报道"），无"信源是 XX，X 月 X 号"机械句式反复（全篇 ≤2 次）；**无"点评："书面标签**（用自然判断句式）。
- **4**：基本友好，仅 1 处机器味残留或个别长段，或信源机械句式出现 2-3 次。
- **3**：有少量机器过渡词或长句影响念读，或信源机械句式反复（>3 次），或残留"点评："标签。
- **2**：明显书面化到念不顺，或多处机器词，或信源机械句式满篇。
- **1**：基本是生硬朗读，机器味浓。
- **注**：资讯播报**允许**第三人称客观陈述、适度结构化过渡——这不是扣分项。"口语化"在此指"TTS 念得顺、不生硬"，不等于"朋友闲聊腔"。

### 7. 节奏 Pacing（取代旧"吸引力 Engagement"维度 · 双层节奏）
- **5**：**双层节奏分明**——快报部分详略起伏（重磅详讲带数据/上下文、次要一句话带过），**今日深读充分展开**（背景+三维分析+反向风险，约占正文 30-40%）；节段切换信号明确；不单调念清单；自然判断处点到为止不拖沓。
- **4**：节奏好，偶有平段或某条篇幅失衡，或深读展开不足（<25%）。
- **3**：信息到位但节奏平淡，多条快报等长等重像念清单，或深读与快报篇幅失衡（深读过短沦为普通条目，或过长挤压快报）。
- **2**：节奏单调，听着累或抓不住重点。
- **1**：无法听完（信息堆砌或机械）。
- **注**：本维度**不考核反问/钩子密度**。资讯简报的节奏靠**双层信息组织**（快报详略 + 深读展开）与详略取舍，不靠闲聊钩子。深读环节质量是本维度与维度 2（Arc）的共同考核点。

### 8. 信息保真 Fidelity（**硬门禁**）
- **5**：所有事实 100% 可回溯源文，核心观点全覆盖，零虚构；点评的判断都能从已陈述事实合理推出，无夸大/无依据预测。
- **4**：可回溯，核心观点覆盖，仅措辞改写无失真。
- **3**：有个别无法核实/轻微夸大的表述。
- **2**：有编造/源文外事实/数据失真。
- **1**：大量虚构。
- **注**：Fidelity ≤3 直接判不合格，无论其它维度多高。

### 9. TTS 友好
- **5**：短分句、断句自然、无密集数字/术语堆砌（数字口语化拆开），分段语义完整。
- **4**：基本友好，仅个别密集串需拆。
- **3**：有几处 TTS 易吞字/读错的长串或生僻堆砌。
- **2**：多处数字/术语堆砌，预期吞字严重。
- **1**：满屏密集信息串，TTS 必崩。

### 10. 元数据 Metadata
- **5**：标题≤35字、**含数字钩子（优先源文事实数字）**、关键词前置、**前10字无英文堆砌**、不带集号前缀；简介200-300字（前100字含核心关键词+具体事实）；标签5-8。
- **4**：达标，仅微瑕（如数字钩子偏弱或前10字含短英文）。
- **3**：长度/关键词/标签数/数字钩子某项不达标。
- **2**：多项不达标。
- **1**：缺失或严重不合格。

## 市场可用门槛（market-ready bar）

- 每个维度 ≥ **4**（10 维：hook / structure / outro / clarity / anti_formulaic / orality / pacing / fidelity / tts_friendly / metadata）
- 总分（10 维平均）≥ **4.0**
- 无任何维度 < 4（即不许有短板）
- **Fidelity 必须零虚构**（Fidelity=5，且 judge 的 `fabricated_claims` 为空）

未达标 → 进入 fix 循环，针对低分维度定向改写。
