# 单次合成 TTS

> `scripts/solo_tts.py` 的实现依据 + agent 理解"为什么全文一次合成"。

## 核心结论：全文一次合成，不拆段

单人独白最该避免的是**分段边界处的音色漂移和节奏断裂**。整段一次合成，音色/语气从头到尾最一致——这是单人播客的生命线。

实测（2026-06-29，solo_tts.py 单次合成）：
- 1786 字 → 83s 合成 → 350s mp3 ✓
- 1835 字 → 65s 合成 → 281s mp3 ✓

前提：**10s 短 ref（避开 voiceclone 大请求限流）+ timeout 足够（~300s）**。这俩满足，~2000 字级别单次合成稳过。

## 为什么不拆段

旧记录（2026-06-25）曾以为"全文一次合成卡死"。根因是 30s 完整 ref（撞 429）或 timeout 太短。
2026-06-27 实测纠正：短 ref + 长 timeout → 单次合成成功。**分段路径已删除**——它是为已消灭的故障预留的过度设计。

## 不要做

- ❌ 不要拆段——单音色一致性是单人播客的生命线。
- ❌ 不要用 30s 长 ref（429）或过短 timeout（超时）。
- ❌ 不要把中文输出路径直接喂 ffmpeg（用无中文 temp 目录 + shutil.copy）。
