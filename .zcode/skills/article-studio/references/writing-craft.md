# 写作工艺强化（writing-craft）

> article-writer agent（DRAFT/FIX）写作工艺真源。与 [phase1-writing.md](phase1-writing.md) 的通用工艺互补——那里讲"跨类型的基本要求"（短句多段、事实保真、反 AI 腔、格式），本文件讲**怎么把文章写得抓人、好读、有记忆点**的强化工艺。

> **读者代表（`agents/reader.md`）按本文件的四块给分**：钩子对应 engagement，节奏对应 reading_experience，金句对应 engagement，获得感对应 takeaway。写手按本文件写，读者代表按本文件审，标准对齐。

## 五块总览

| 块 | 解决的问题 | 核心动作 | 对应读者代表维度 |
|----|----------|---------|----------------|
| 一、开头钩子 | 第一段就想划走 | 6 种钩子模式，按类型选 | engagement |
| 二、去 AI 腔 / 口语化 | 像机器写的 | 软 AI 腔识别 + 口语化改写 | reading_experience |
| 三、阅读节奏 / 排版 | 读不下去 | 长短句交替、加粗克制、列表化、视觉呼吸 | reading_experience |
| 四、金句 / 记忆点 | 读完啥也没记住 | 每 200-300 字一个记忆点 | engagement + takeaway |
| 五、结尾 CTA | 读完就走、不互动 | 文末引导留言/转发/关注 | engagement |

## 一、开头钩子

公众号文章的前 3 段决定读者走不走。**不要平铺直叙开场**（"今天我想聊聊 X"、"随着 Y 的发展"——这是典型劝退开头）。

### 6 种钩子模式

| 钩子 | 怎么写 | 适用类型 | 例子 |
|------|-------|---------|------|
| **提问** | 戳读者的真实困惑/痛点，让他点头"对对对" | howto / opinion | "你是不是也这样：每次想写点东西，打开编辑器，光标闪了半小时，还是一个字没动？" |
| **反差** | 把两个对立的事实/感受并排，制造认知冲突 | opinion / review / news | "半年前我还觉得这工具是智商税，现在它是我每天打开次数最多的 App。" |
| **场景代入** | 一帧画面/一句对话开场，把读者拉进现场 | story / profile | "凌晨三点，我盯着屏幕上第 17 次失败的报错，差点把键盘砸了。" |
| **金句** | 一句有传播感的话开场，先声夺人 | opinion / profile | "真正的效率，从来不是做得更快，而是少做废动作。" |
| **数据冲击** | 一个反直觉/有冲击力的数字 | news / review | "2026 年上半年，有 47 个大模型扎堆发布——平均每 4 天一个。" |
| **悬念** | 抛出一个未解的问题/未知的结局 | story / profile | "那个把公司卖掉去种地的产品经理，三年后我再见他时，他说的第一句话让我愣住了。" |

### 按类型推荐钩子

| `article_type` | 首选钩子 | 避免 |
|---------------|---------|------|
| `opinion` | 反差 / 金句 | 平铺"今天聊 X" |
| `howto` | 提问（戳痛点） | 直接列步骤 |
| `story` | 场景代入 / 悬念 | 先铺背景 |
| `news` | 数据冲击 | "本周发生了很多事" |
| `profile` | 场景代入 / 金句（人物原话） | 简历式介绍 |
| `review` | 反差（用前用后）/ 提问（痛点共鸣） | 硬广式开场 |

### 反例（禁止这样开头）

❌ "随着 AI 的快速发展，越来越多的工具开始涌现……"（空洞宏大叙事）
❌ "今天我想和大家聊聊 XX 这个话题。"（平铺直叙）
❌ "在这个信息爆炸的时代……"（陈词滥调）

✅ 改写思路：把抽象开场换成一个**具体的画面/数字/提问**，让读者第一句就"进去"。

## 二、去 AI 腔 / 口语化

机器预检（`validate_content_quality.py`）能查出**硬 AI 腔**（blocklist 短语：首先/其次/综上所述/值得等等）。但还有一类**软 AI 腔**机器查不出、读者一眼能感觉到——本块解决这个。

### 软 AI 腔的 5 个信号（自查）

| 信号 | 表现 | 为什么像 AI |
|------|------|-----------|
| **句式雷同** | 连续多段都是"X 是 Y。Z 也是 Y。W 还是 Y。"的并列结构 | 机器爱排比，人不会这么整齐 |
| **过度对仗** | 每句都追求对工整："不仅要 X，更要 Y""一方面 X，另一方面 Y" | 人说话有参差，不会句句对仗 |
| **排比三连** | 动不动"可以 X，可以 Y，可以 Z"三连以上 | 排比是 AI 的舒适区，滥用显假 |
| **关联词滥用** | 不仅/而且/因此/然而/与此同时 密度过高 | 连接词堆叠是书面语病，口语里很少 |
| **抽象宏大叙事** | "赋能/抓手/生态/闭环/全面/显著提升"等抽象大词 | 说具体事的人不会用这些词 |

### 口语化改写手法

1. **加语气词与停顿**：把"该工具显著提升了效率"→"用它之后，我确实快了不少"。口语允许"其实""说实话""老实讲"这类软词（但别滥用）。
2. **短句断句**：一个长句拆成两三个短句。"虽然 X 但是 Y 因此 Z"→"X。不过 Y。所以 Z。" 手机上短句更顺。
3. **书面语换人话**：
   - "进行了一次深度的探讨" → "好好聊了聊"
   - "实现了效率的显著提升" → "快了不少"
   - "该方案具备可行性" → "这招能成"
4. **第一人称代入**：把"人们通常认为"换成"我自己的感觉是"（opinion/review/howto 尤其重要）。把无主语句补上"你/我/它"。
5. **删掉空连接词**：很多"因此/所以/综上所述"删掉，句子更干净，逻辑还在。

### 自查流程（DRAFT 写完后做一遍）

逐段读，问自己：
- 这段连续 3 句以上是不是同一个句式？→ 打散
- 是不是每句都在追求对仗工整？→ 故意写"歪"一点
- 抽掉所有"不仅/而且/因此/然而"，句子还通吗？→ 通就抽掉
- 有没有"赋能/抓手/生态/闭环"这类大词？→ 换成具体说法

## 三、阅读节奏 / 排版（手机端）

公众号文章 80% 在手机上读。手机端阅读的核心是**视觉呼吸**——读者眼睛要有地方歇。

### 5 条节奏规则

| 规则 | 要求 | 为什么 |
|------|------|-------|
| **单段 ≤120 字** | 超过就拆段（已有约束，必须守） | 手机上一段超过 6-7 行就是一堵墙 |
| **长短句交替** | 避免"全短句"或"全长句"。一段里允许 1 个长句 + 几个短句 | 全短句像电报，全长句像论文，交替才像人说话 |
| **关键句加粗（克制）** | 每 200-300 字最多 1 处加粗，只加粗"读者扫读时该停下来的那句" | 满屏加粗 = 没有加粗；滥用加粗反而显 AI |
| **列表化步骤/枚举** | 有 3 个以上并列项（步骤/清单/对比）就用列表，不用逗号堆在一个段落 | 手机上列表比"步骤一……步骤二……步骤三……"的段落好读十倍 |
| **小标题断节奏** | 每 300-500 字一个小标题（`##`/`###`），给读者"喘气点" | 无小标题的长文，中途流失率高 |

### 排版反例

❌ 一段 300 字不分段（一堵墙）
❌ 全文没有一个加粗（读者扫读抓不住重点）
❌ 每段都加粗好几句（加粗失效）
❌ 步骤写成"第一步你需要……第二步你需要……第三步你需要……"的长段落（应该用列表）

### 扫读测试（写完后做）

假装自己是手机上快速划屏的读者，只看：标题 + 加粗句 + 小标题 + 列表项。**如果光看这些就能大致get到文章在讲什么**，排版就过关了。

## 四、金句 / 记忆点

读者读完一篇文章，能记住的往往只有 1-3 句话。**写手的任务之一是主动制造这些记忆点**，而不是让它们随机出现。

### 密度要求

**每 200-300 字至少 1 个记忆点。** 一篇 1500 字的文章，至少要有 5-6 个记忆点候选。

### 记忆点的 5 种类型

| 类型 | 怎么写 | 适用类型 |
|------|-------|---------|
| **对比金句** | 把两个对立概念并排，制造张力 | opinion / review |
| | "别人在卷参数，他在卷体验。" | |
| **重新定义** | 给一个常见词下个反常规的定义 | opinion / howto |
| | "所谓效率，不是做得更快，而是敢于不做。" | |
| **反常识** | 说一个违反直觉、但细想有理的点 | opinion / news |
| | "真正拖慢你的，往往不是工具差，而是工具太多。" | |
| **具象比喻** | 把抽象概念用一个画面说清楚 | 全类型 |
| | "它像瑞士军刀——功能多，但你不会拿它切牛排。" | |
| **场景/细节** | 一个具体到能看见画感的细节 | story / profile |
| | "他说话时，手指无意识地摩挲着那个磨掉漆的保温杯。" | |

### 按类型选记忆点

| `article_type` | 首选记忆点类型 |
|---------------|--------------|
| `opinion` | 对比金句 / 反常识 / 重新定义 |
| `howto` | 反常识（戳破常见误区）/ 具象比喻 |
| `story` | 场景细节 / 对比金句（结尾升华） |
| `news` | 数据型记忆点（盘点文本身就有）/ 反常识点评 |
| `profile` | 人物原话金句 / 场景细节 |
| `review` | 具象比喻（形容体验）/ 对比金句 |

### 金句写法（怎么写出"能转发"的句子）

1. **对比结构**：A 是 X，B 是 Y（A/B 对立）。"工具决定下限，习惯决定上限。"
2. **重新定义**："X 不是 Y，而是 Z。" 把常见认知翻转。
3. **具象化**：把抽象形容词换成具体画面。"很好用" → "我用了一周就回不去了。"
4. **短而有力**：金句要短（≤20 字最佳），一句话能单独截图传播。避免长定语从句。

### 反例（假金句）

❌ "在这个快速变化的时代，我们需要不断学习，才能跟上步伐。"（正确的废话，谁都不会转发）
❌ "AI 是一把双刃剑，既带来机遇，也带来挑战。"（陈词滥调）

✅ 改写：把"正确的废话"换成**一个具体、反常规、能引发点头或争论**的说法。

## 五、结尾 CTA（行动召唤）

文章正文结束后，**文末要加一段 CTA**（Call To Action，行动召唤），引导读者留言、转发、关注。位置在正文结尾之后、AI 生成声明（红线 6）之前。

### CTA 三件套

一段 CTA 通常引导三个动作（不必三条全要，按文章情况选）：

| 动作 | 怎么写才有效 |
|------|------------|
| **留言** | 给一个**具体的话题切入点**，不要泛泛"欢迎留言"。让读者有明确的话可接，最好和文章主题相关（例：「你用 AI 做过视频吗？拼接、AI 生成、代码动画，你选哪个？」） |
| **转发** | 精准定位转发对象，不是"转发给朋友"，而是"转发给同样在折腾 X 的朋友"——让转发有明确去处 |
| **关注** | 给关注一个理由（关注了能看到什么），配合连载文章尤其有效 |

### 写法要点

- **承接正文，别硬接**：CTA 开头顺着正文主题自然引出，不要突兀地开始喊"关注点赞转发"。可以先点一句和文章相关的延续想法，再转到互动引导。
- **每句给一个理由**：不是"欢迎留言"，而是"欢迎留言**聊聊**"；不是"请关注"，而是"关注我，**跟进后续**"——理由让动作更有说服力。
- **不堆砌、不卑微**：三句话三个动作（留言/转发/关注），节奏清楚即可，不要反复恳求。公众号读者反感"求关注求转发"的乞讨腔。
- **不做无法兑现的承诺**：别用"下期拆 X"这种内容预告，除非确实打算写——预告了不写会失信。要引导关注，用"跟进后续"这类不绑定具体内容的表述更稳妥。

### 反例（禁止这样写 CTA）

❌ "如果觉得文章有用，记得点赞关注转发哦！你们的支持是我最大的动力！"（乞讨腔，三连堆砌，无理由）
❌ "欢迎在评论区留言，说说你的看法。"（泛泛，读者不知道说什么）
❌ "下一篇我会拆解视频生成的细节，敬请关注！"（做了具体预告，万一不写就失信）

✅ 改写：给具体话题 + 给动作理由，语气平实不卑微。

### 标准结构（参考）

> [承接正文一句] + [留言引导，带具体话题] + [转发引导，带精准对象] + [关注引导，带理由]
>
> （然后是 AI 生成声明 `> 本文由...AI 辅助...`）

### 与红线 6（AI 声明）的顺序

文末固定顺序：**正文 → CTA → AI 生成声明**。CTA 在声明之前（声明永远是文末最后一行）。

## 与通用工艺（phase1-writing.md）的关系

本文件是 phase1-writing.md 的**强化补充**，不替代它。写手 DRAFT 时：

1. 先守 phase1-writing.md 的**通用底线**（短句多段、事实保真、反硬 AI 腔、格式规范）
2. 再叠加本文件的**强化工艺**（钩子开场、软 AI 腔自查、节奏排版、主动制造记忆点）

通用底线不过关 → 机器预检/内容主编就会拦下来。强化工艺不过关 → 读者代表会拦下来（engagement/reading_experience 低分）。
