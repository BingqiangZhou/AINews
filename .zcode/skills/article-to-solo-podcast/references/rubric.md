# 单人播客脚本评分 Rubric（"市场可用"的定义）

> `solo-script-judge` 的评分真源。10 维，每维 1–5 分。打分要严格、要给证据（引用脚本原文 + 源文对照）。
> 目的：把"市场可用"从主观感觉变成可量化、可迭代的门槛。

## 评分维度

### 1. 钩子 Hook / Cold Open（前 ~20 秒）
- **5**：cold open 直接入题（反问/痛点/悬念/具体场景），15 秒内落钩，零问候套话；落钩后自然报家门（品牌+集号+主题），不生硬。
- **4**：有效 cold open，吸引人，仅微瑕。
- **3**：有钩子意图但平淡，或报家门位置生硬（如开头立刻报家门未先落钩）。
- **2**：开场啰嗦/慢热，前 15 秒抓不住人。
- **1**：问候式/公式化开场（"大家好/欢迎/今天聊"先于内容）。

### 2. 结构 Arc
- **5**：hook→promise→body→climax→CTA 完整且自然，要点全部结论先行。
- **4**：结构清晰，偶有一处过渡略硬。
- **3**：大体有结构，但缺承诺/高潮或要点不结论先行。
- **2**：流水账，无明显弧线。
- **1**：结构混乱/断裂。

### 3. 结尾 Outro
- **5**：清晰回顾要点 + 一个有力 takeaway + 一个明确 CTA（订阅/关注）或思考题 + 自然告别，与正文呼应，零套话。
- **4**：收尾有效，仅微瑕。
- **3**：有收尾但偏平淡/套路，或缺回顾、缺 takeaway/CTA。
- **2**：收尾仓促，或套话（"今天就聊到这里""感谢收听""我们下期再见"品牌定式）。
- **1**：无收尾/硬断。

### 4. 趣味性 Fun
- **5**：比喻/类比 ≥2、口语夸张/生动表达 ≥2、有人设观点、用具体场景落地；听着不困且不油腻，趣味服务内容。
- **4**：有趣味元素，密度/自然度偶有不足。
- **3**：少量趣味，偏"讲知识"，枯燥段较多。
- **2**：几乎无比喻/生动表达，干瘪。
- **1**：满屏说教/念稿，无任何趣味。

### 5. 反套路 Anti-formulaic
- **5**：零 AI 套路——无三段及以上排比收尾、无路标编号、无套式连接词反复、无"反认知/反直觉"等 AI 高频词；句式错落。
- **4**：基本无套路，仅 1 处 borderline。
- **3**：出现 1-2 处明显套路（如一处排比收尾、或一处路标编号）。
- **2**：多处套路，AI 味明显。
- **1**：通篇模板化/工整排比/路标化，AI 味浓。

### 6. 口语化 Orality
- **5**：全程第一人称、短句、口语过渡，零机器味词，无 Markdown，段落全 ≤80 字。
- **4**：口语化好，仅 1 处机器味残留或个别长段。
- **3**：偏书面，有少量机器过渡词或长句。
- **2**：明显书面化/朗读感，多处机器词。
- **1**：基本是文章朗读，机器味浓。

### 7. 吸引力 Engagement
- **5**：反问/类比密集且自然，中段每 3-5 分钟一个新钩子，有真正高潮/洞察。
- **4**：吸引人，节奏好，偶有平段。
- **3**：信息到位但偏"讲知识"，缺张力。
- **2**：枯燥，像念稿。
- **1**：无法听完。

### 8. 信息保真 Fidelity（**硬门禁**）
- **5**：所有事实 100% 可回溯源文，核心观点全覆盖，零虚构。
- **4**：可回溯，核心观点覆盖，仅措辞口语化无失真。
- **3**：有个别无法核实/轻微夸大的表述。
- **2**：有编造/源文外事实/数据失真。
- **1**：大量虚构。
- **注**：Fidelity ≤3 直接判不合格，无论其它维度多高。

### 9. TTS 友好
- **5**：短分句、断句自然、无密集数字/术语堆砌，分段语义完整。
- **4**：基本友好，仅个别密集串需拆。
- **3**：有几处 TTS 易吞字/读错的长串或生僻堆砌。
- **2**：多处数字/术语堆砌，预期吞字严重。
- **1**：满屏密集信息串，TTS 必崩。

### 10. 元数据 Metadata
- **5**：标题≤35字、**含数字钩子（优先源文事实数字）**、关键词前置、**前10字无英文堆砌**、不带集号前缀；简介200-300字（前100字含核心关键词+具体事实）；标签5-8。
- **4**：达标，仅微瑕（如数字钩子偏弱或前10字含短英文）。
- **3**：长度/关键词/标签数/数字钩子某项不达标。
- **2**：多项不达标。
- **1**：缺失或严重不合格。

## 市场可用门槛（market-ready bar）

- 每个维度 ≥ **4**（10 维：hook / structure / outro / fun / anti_formulaic / orality / engagement / fidelity / tts_friendly / metadata）
- 总分（10 维平均）≥ **4.0**
- 无任何维度 < 4（即不许有短板）
- **Fidelity 必须零虚构**（Fidelity=5，且 judge 的 `fabricated_claims` 为空）

未达标 → 进入 fix 循环，针对低分维度定向改写。

```json
{
  "round": 1,
  "market_ready": false,
  "overall": 3.9,
  "dimensions": {
    "hook": { "score": 5, "evidence": "引用脚本原文片段", "gaps": [] },
    "structure": { "score": 4, "evidence": "...", "gaps": ["高潮段偏弱"] },
    "outro": { "score": 4, "evidence": "结尾回顾要点+订阅引导", "gaps": [] },
    "fun": { "score": 4, "evidence": "比喻'像把说明书读给朋友听'+口语'快得离谱'+人设'最先扔了'", "gaps": [] },
    "anti_formulaic": { "score": 5, "evidence": "无排比收尾/无路标编号/连接词多样", "gaps": [] },
    "orality": { "score": 3, "evidence": "第二段出现'首先'", "gaps": ["去机器词：第2/5段"] },
    "engagement": { "score": 4, "evidence": "...", "gaps": [] },
    "fidelity": { "score": 5, "evidence": "逐条对照源文", "fabricated_claims": [] },
    "tts_friendly": { "score": 3, "evidence": "'28到99秒中位40到50秒'密集数字串", "gaps": ["拆第6段数字串"] },
    "metadata": { "score": 4, "evidence": "...", "gaps": [] }
  },
  "fix_directives": [
    { "target": "第2段", "problem": "机器词'首先'", "fix": "改为'先说速度这块'这类口语过渡" },

```

`fix_directives` 必须具体到位置 + 问题 + 改法，供 `solo-scriptwriter` 的 fix 模式定向改写。
